Слайд 1 — Заголовок
Техническая часть: Система управления рестораном

Заметки:
Краткое вступление — цель презентации: раскрыть архитектуру, стек технологий, организацию данных и способы развёртывания.

Слайд 2 — Архитектура приложения
- Desktop WPF приложение (MVVM)
- Проектная структура:
  - `RestaurantSystem.Core` — модели, общие типы, результатные обёртки
  - `RestaurantSystem.Data` — EF Core DbContext, миграции/инициализация, сиды
  - `RestaurantSystem.UI` — WPF View + ViewModel, конвертеры, ресурсы
- Внедрение зависимостей через Generic Host / ServiceCollection

Заметки:
Покажите простую диаграмму: UI <-> ViewModels <-> Сервисный слой <-> DbContext (SQLite).

---

Слайд 3 — Технологический стек
- Язык и рантайм: C# / .NET (целевой фреймворк проекта)
- UI: WPF, XAML, MaterialDesignInXaml / MahApps (для оформления)
- ORM: Entity Framework Core с локальной SQLite (файл `RestaurantSystem.db`)
- Паттерны: MVVM, DI (Microsoft.Extensions.DependencyInjection), репозитории/сервисы
- Сценарии тестирования: Unit-тесты (xUnit / MSTest) в `tests/`

Заметки:
Упомянуть преимущества: быстрый UI dev (WPF), лёгкая локальная БД для демо и тестов (SQLite).

Слайд 4 — Модель данных и сиды
- Основные сущности: Dish, Order, OrderItem, Table, Reservation, User
- Dish: имя, категория (Enum), цена, время приготовления, теги, аллергены
- Инициализация БД:
  - `DbContext.EnsureCreated()` при старте
  - Сеялка (seed) добавляет таблицы и стандартное меню, если таблицы пусты
- Файл подключения: `appsettings.json` -> `DefaultConnection` (Data Source=RestaurantSystem.db)

Заметки:
Объяснить, где в коде добавлена логика сидирования (DependencyInjection/InitializeDatabaseAsync).

Слайд 6 — Сборка, тесты и развёртывание
- Сборка: `dotnet build` (решение в `scripts/RestaurantSystem.sln`)
- Тесты: Unit-тесты в `tests/` — запуск через `dotnet test`
- Скрипты развёртывания: `scripts/publish-app.ps1`, `create-installer.ps1` и инструкции в `BUILD_INSTALLER.md`
- Установка: проект содержит Inno Setup скрипт `setup/setup.iss` для создания установщика

Заметки:
Пояснить процесс: билд -> публикация -> создание установщика -> дистрибуция. Упомянуть проверку целевых runtime и зависимостей.

Слайд 7 — Риски, улучшения и дальнейшие шаги
- Риски / ограничения:
  - Неполная локализация (строки в XAML) — сейчас нельзя переключать языки динамически
  - Локальная SQLite подходит для малого масштаба; миграция на серверную БД потребует настроек
- Предложения по улучшению:
  - Вынести строки в `.resx` и добавить переключение культур
  - Добавить интеграционные тесты и сценарные UI-тесты (например, WinAppDriver)
  - Добавить CI-пайплайн (build + test + pack + release)
- Что можно показать на демо:
  - Сборка приложения, автоматическое сидирование меню, пример создания заказа и брони, экспорт/отчёт по продажам

Заметки:
Заключение: кратко повторите цели и преимущества архитектуры, и обозначьте ближайшие шаги для промышленной зрелости.
